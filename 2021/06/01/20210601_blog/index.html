Hexo + Github Pages 搭建个人博客

Lau

2021-06-01 19:25:00

本文主要描述了如何最快的通过 Hexo 和 Github 搭建博客，以及在搭建过程中遇到的问题。
在配置博客之前，需自行安装 Git，Node.js 环境以及 Github 账号，这里暂不描述。

  • Github Pages
  • Hexo 博客框架
  • 主题选择
  • 部署 Github

Github Pages

Github Pages 本身是 Github 提供的博客服务。

具体操作如下

  1. 首先创建一个 Github 仓库，该仓库即用于自己的博客仓库。

  1. 仓库名设置为 xxx.github.io，并勾选上 README.md。

注意：xxx 是 Github 的用户名（非昵称），且注意大小写，默认都是小写，否则访问的域名将不是 xxx.github.io ，而是 xxx.github.io/yyy.github.io 的子级页面。

  1. README.md 是说明文档，对该项目的简要描述。
  2. 访问 xxx.github.io ，当前默认打开应该为 README.md 的内容。

Hexo

Hexo 是我们当前选择的一个博客框架，在安装 Hexo 前，需安装 Node.js 和 Git 环境，全部安装完毕后，具体操作如下。

全局安装 Hexo

3.右键该文件夹，选择 Git Bash Here 打开命令行。

在浏览器输入 http://localhost:4000/ 即可查看本地博客项目。

4.如何写一篇博客。

在 /source/_posts/ 文件夹下新建 xx.md 文件，里面即为博客内容。通过 hexo g 重新生成网页，hexo s 重新启动服务，即可看到多了一篇博客。

主题选择

这里我们选择Next主题，Next支持丰富的主题样式，具体操作如下。

  1. https://hexo.io/themes/ 选择自己喜欢的主题，下载该主题的压缩包。
  2. 到 /themes/ 目录下，将解压后的文件夹放到该目录下，并将该主题的文件夹名修改为 next。
  3. 修改配置文件。
  • 打开 _config.yml 文件
  • 找到 theme:
  • 将默认的 landscape 修改为 next ，注意：冒号和后面的内容需要有空格，否则会编译失败，即 theme: next，冒号和next之间有一个空格，所有的配置项都如此。
  • 找到 # Site，添加博客名称，作者名字等。
  • 找到 # URL, 填入 url。比如 url: https://liux4ever.github.io
  • 其它配置信息根据需求自行修改。
